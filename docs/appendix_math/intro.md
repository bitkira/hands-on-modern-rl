# 附录 E：强化学习的数学基础

本附录不是按算法来堆公式，而是按数学学习的自然顺序来组织：先认识数学对象，再学习运算和方程，然后进入概率、随机估计、优化，最后回到强化学习中的完整推导。

读者不需要一开始就理解 PPO、GAE 或 DPO。更合理的路线是：先知道标量、向量、矩阵、随机变量、期望、导数、梯度、熵和 KL 分别在说什么，再看它们如何组合成强化学习公式。

## 学习路线

数学节奏可以概括为：

数学对象 → 线性运算 → 概率与期望 → 随机估计 → 递推方程 → 优化与梯度 → 分布距离 → 强化学习完整公式。

## 一个贯穿例子

后面几节会反复使用一个最小的两状态例子。环境有两个状态 $s_1, s_2$：

- 在 $s_1$，智能体获得奖励 $2$，然后转到 $s_2$。
- 在 $s_2$，智能体获得奖励 $1$，然后转回 $s_1$。
- 折扣因子取 $\gamma = 0.5$。

如果设两个状态的价值分别为 $v_1, v_2$，那么直觉上：

$$
\begin{aligned}
v_1 &= 2 + 0.5v_2, \\
v_2 &= 1 + 0.5v_1.
\end{aligned}
$$

这个例子会在不同数学章节中扮演不同角色：

- 在线性代数中，它是一个二元线性方程组。
- 在概率统计中，它是“即时奖励 + 下一状态期望价值”。
- 在随机估计中，它可以通过采样轨迹来近似。
- 在优化中，它会变成价值网络或策略网络的训练目标。
- 在信息论中，它会连接到策略分布、探索和更新约束。

只要能把复杂公式翻译回这个例子，数学就会从障碍变成工具。

## 如何使用左侧目录

左侧目录现在分成四个数学模块。它不是要求你一次读完，而是提供三种使用方式：

1. **系统学习。** 从 E.1.1 开始顺序阅读，适合想补数学基础的读者。
2. **按问题查阅。** 看不懂贝尔曼矩阵形式，就读 E.1.2；看不懂 GAE，就读 E.2.4 和 E.3.5；看不懂 KL 约束，就读 E.4.2 和 E.4.3。
3. **考前/复习。** 每个模块最后都有“公式与练习”，适合快速检查是否真正理解。

如果你觉得侧栏条目很多，建议先只读每个模块的第一篇：

- [E.1.1 标量、向量与矩阵](./linear-algebra-basics)
- [E.2.1 概率、条件概率与期望](./probability-basics)
- [E.3.1 导数、梯度与链式法则](./calculus-basics)
- [E.4.1 自信息、熵与探索](./information-basics)

这四篇读完后，再根据正文遇到的公式回到对应专题。
